# Skillify — 从对话提炼 Skill

你的任务是分析当前对话中执行过的流程，将其总结为可复用的 skill 文件，并用 `skill_save` 保存。

## 执行步骤

### 第 1 步：分析当前对话

回顾完整的对话历史，识别：

- 执行了什么**可重复**的流程（区别于一次性操作）
- 流程的**输入参数**是什么（文件路径、任务描述等）
- 有哪些**关键步骤**（按顺序）
- 每步的**成功标准**是什么
- 用到了哪些**工具**（bash / read_file / write_file / glob_search 等）
- 用户在哪些地方**纠正或调整**了你的行为（这些偏好需要写入 skill）

### 第 2 步：向用户确认

用**普通文本**问用户以下问题（一次问完，不要分多轮）：

1. **Skill 名称**：建议一个英文连字符名称，例如 `code-review-pr`
2. **一句话描述**：这个 skill 做什么
3. **何时使用**：什么情况/触发词应该自动调用它（例如："用户说'帮我 review PR'时"）
4. **参数**：有无需要传入的参数（如 PR 编号、文件路径）
5. **保存位置**：
   - `skills_bundled`（内置，所有项目可用）
   - 用户目录 `~/.shell-agent/skills/`（个人，跨项目）

根据用户回复调整内容。

### 第 3 步：生成 Skill 内容

按以下格式生成 SKILL.md 内容，**先在回复中展示给用户确认**，再保存：

```markdown
---
name: {{skill-name}}
description: {{一句话描述，20字以内}}
when_to_use: {{详细说明何时自动调用，包含触发词示例。以"当用户..."开头}}
arguments:
  - {{参数名}}: {{参数说明}}
---

# {{Skill 标题}}

{{2-3句话介绍这个 skill 的目标和适用场景}}

## 输入
- `${{参数名}}`：{{参数说明}}（若无参数则省略本节）

## 目标
{{明确的完成标准：不只是"完成任务"，而是"产出 X、通过 Y 验证"}}

## 步骤

### 1. {{步骤名}}
{{具体操作，包含命令示例}}

**成功标准**：{{判断此步骤完成的具体条件}}

### 2. {{步骤名}}
...

## 注意事项
- {{用户在参考会话中纠正过的行为}}
- {{容易出错的地方}}
- {{硬性约束（必须/不能做的事）}}
```

**格式规则**：

- `when_to_use` 是**最重要的字段**：模型靠它决定是否自动调用，必须含有触发词示例
- `description` 不超过 20 字
- 每个步骤必须有**成功标准**，否则模型不知道何时继续下一步
- 有参数时用 `$arg_name` 占位，无参数时省略 `arguments` 和 `## 输入` 节

### 第 4 步：保存

用户确认后，调用 `skill_save`：

- 名称：用户确认的 skill 名称
- 内容：完整的 markdown（含 YAML frontmatter）

保存成功后告知用户：

- 保存路径
- 如何调用：`skill_use {{skill-name}}`
- 可以直接编辑 `.md` 文件进行微调

## 参考示例

以下是一个**好的** skill 片段，注意 `when_to_use` 的写法：

```markdown
---
name: commit-and-pr
description: 提交代码并创建 Pull Request
when_to_use: 当用户说"提交代码"、"创建 PR"、"commit 并推送"、"帮我提 PR"等类似语句时使用。适用于完成功能开发后的标准化提交流程。
arguments:
  - title: PR 标题（可选，默认从 git diff 自动生成）
---
```
